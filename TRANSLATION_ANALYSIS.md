### Общая проблема обоих переводов

Строки вида `` [`object`](url) `` — в ряде мест ссылка ломается:
- GPT: полностью теряет часть строки (`[`object`Template pages...`)
- bitrixgpt: вставляет пробел (`[`object`] (url)`)

Источник — скорее всего Protector: `` [`object`](url) `` — это ссылка где текст содержит inline-код. Если сначала защищается `` `object` `` (inline-код), остаётся `[PROTECTED](url)`, затем ссылка защищается нормально. Но если порядок другой или ссылка уже защищена целиком — могут возникать артефакты. Требует проверки Protector.

## Найденные проблемы

### 1. Русский текст ссылок не переводится

**Строки:** 221, 387

Protector защищает ссылки целиком `[text](url)` — LLM не видит текст.

```
[ниже](#deal)                              →  остаётся "[ниже](#deal)"
[о пользовательских полях](./index.md)    →  остаётся с русским текстом
```

**Решение:** изменить protector — защищать только URL, текст ссылки оставлять для LLM.

---

### 2. JSON-значения внутри блоков кода не переводятся

**Строки:** 157, 178–179, 188, 201

Блоки ` ``` ` защищаются до LLM — промпт говорит "переводи строки в JSON", но LLM их не получает.

```json
"TITLE" "Новая сделка #1"           →  остаётся русским
"COMMENTS": "[B]Пример комментария"  →  остаётся русским
"UF_CRM_...": "Привет мир!"          →  остаётся русским
```

**Сложность:** нужно различать значения-идентификаторы (`"CALLBACK"`, `"COMPLEX"`) и текстовые строки. Сейчас защищается весь блок.
**Приоритет:** низкий — эти строки являются примерами, не влияют на функциональность документации.

---

### 3. "воронка" переведено как "Sales Funnel" вместо "pipeline"

**Строка:** 248

```
CATEGORY_ID | Sales Funnel.   ←  должно быть "Pipeline"
```

Модель видит CRM-контекст и выбирает "Sales Funnel". Поле `CATEGORY_ID` — это технический термин для воронки/пайплайна сделок.

**Решение:** уточнить в промпте: `воронка` (без уточнений "продаж"/"CRM") = pipeline.
