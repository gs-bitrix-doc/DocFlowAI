# DocFlow AI

Инструмент для автоматического перевода `.md` файлов документации с сохранением структуры.

## Как запустить

**1. Установить зависимости:**
```bash
uv add requests python-dotenv click mistune
```

**2. Заполнить `.env`** (скопировать из `.env.example`):
```env
API_KEY=your_api_key_here
BASE_URL=https://your-bitrix-gpt-endpoint
MODEL=your_model_name
```

**3. Запустить:**

```bash
# Перевести один файл
python main.py input/crm-deal-get.md

# Перевести всю папку
python main.py --dir input/

# Проверить пайплайн без перевода (без API ключа)
python main.py input/crm-deal-get.md --dry-run
python main.py --dir input/ --dry-run
```

Результат сохраняется в `output/` с суффиксом `.en.md`:
```
input/crm-deal-get.md → output/crm-deal-get.en.md
```

## Пайплайн

```
Файл → Parser → Protector → Translator → Protector → Validator → Writer
```

## Модули

### `src/parser.py` — MarkdownParser

Отвечает за чтение `.md` файла и валидацию входных данных.

**Поля:**
- `file_path` — путь к файлу (`Path`)
- `filename` — имя файла с расширением (`crm-deal-get.md`)
- `stem` — имя файла без расширения (`crm-deal-get`)

**Методы:**
- `__init__(file_path)` — принимает путь к файлу
- `validate()` — проверяет что файл существует и имеет расширение `.md`
- `read() → str` — читает и возвращает содержимое файла

**Не делает:** не парсит структуру markdown, не переводит, не сохраняет.

---

### `src/writer.py` — MarkdownWriter

Отвечает за сохранение переведённого текста в папку `output/`.

**Поля:**
- `output_dir` — путь к папке сохранения (`Path`)

**Методы:**
- `__init__(output_dir)` — принимает путь к папке `output/`
- `save(content, stem) → Path` — сохраняет текст как `{stem}.md`, возвращает путь к файлу

**Не делает:** не читает файлы, не переводит, не трогает структуру markdown.

---

### `src/protector.py` — MarkdownProtector

Отвечает за защиту markdown-структуры от изменений при переводе.
Заменяет блоки на плейсхолдеры `[[PROTECTED_N]]` перед переводом и восстанавливает их после.

**Поля:**
- `_blocks: dict` — словарь `{ "[[PROTECTED_0]]": "оригинальный блок" }`, сбрасывается при каждом `protect()`
- `_PATTERNS: list` — упорядоченный список regex паттернов

**Методы:**
- `__init__()` — инициализация
- `protect(content: str) → str` — заменяет блоки на плейсхолдеры
- `restore(content: str) → str` — восстанавливает плейсхолдеры обратно

**Порядок защиты (важен — сначала многострочные блоки, потом инлайн):**

| Приоритет | Что защищаем | Пример |
|---|---|---|
| 1 | Frontmatter | `---\ntitle: ...\n---` |
| 2 | Битрикс-блоки | `{% note %}...{% endnote %}` |
| 3 | Названия полей API | `**TYPE_ID**`, `**CATEGORY_ID**` |
| 4 | Якоря | `{#deal}`, `{#anchor-name}` |
| 5 | Блоки кода | ` ```js\ncode\n``` ` |
| 6 | Картинки | `![alt](url)` |
| 7 | Ссылки | `[текст](url)` |
| 8 | Инлайн-код | `` `crm.deal.get` `` |

**Не делает:** не читает и не сохраняет файлы, не переводит.

---

### `src/translator.py` — Translator

Отвечает за отправку текста в LLM (Bitrix GPT) и получение перевода.

**Поля:**
- `api_key` — ключ API
- `base_url` — адрес API
- `model` — название модели
- `prompt` — системный промпт с инструкциями перевода

**Методы:**
- `__init__(api_key, base_url, model, prompt)` — инициализация
- `translate(content: str) → str` — отправляет текст на перевод, возвращает результат

**Не делает:** не читает файлы, не работает с плейсхолдерами, не сохраняет результат.

---

### `src/validator.py` — StructureValidator

Отвечает за проверку сохранности структуры документа после перевода.
Сравнивает оригинальный и переведённый текст, выявляет расхождения.

**Методы:**
- `__init__()` — инициализация
- `validate(original: str, translated: str) → ValidationResult` — запускает все проверки
- `_extract(content: str) → dict` — извлекает структурные элементы из текста

**Что проверяет:**

| Проверка | Тип | Описание |
|---|---|---|
| Плейсхолдеры восстановлены | error | `[[PROTECTED_N]]` не должно остаться в переводе |
| Количество заголовков | error | число `##` совпадает с оригиналом |
| Количество таблиц | error | число `#\|` и `\|#` совпадает |
| Строки таблиц | error | число `\|\|` совпадает |
| Блоки кода | error | число ` ``` ` совпадает |
| Битрикс-блоки | error | число `{% %}` совпадает |
| Словарь терминов | warning | русские термины из словаря не остались в переводе |

**Результат — `ValidationResult` (dataclass):**
- `is_valid: bool` — прошла ли валидация
- `errors: list[str]` — критичные проблемы (структура сломана)
- `warnings: list[str]` — некритичные (стоит проверить вручную)

**Поведение в пайплайне:** при ошибках файл всё равно сохраняется, но в лог пишется предупреждение.

**Не делает:** не читает файлы, не переводит, не сохраняет.
