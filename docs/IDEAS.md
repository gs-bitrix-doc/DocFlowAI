# Идеи и планы

## DocFlow Web

Сейчас пайплайн — это инструмент, который запускается из командной строки. Web платформа решение проблемы и боли перевода.

### Как выглядит работа на практике

Техпис вносит правки в русскую документацию и публикует их в репозиторий — как обычно. Дальше всё происходит без его участия: сервис замечает изменение, запускает перевод и через несколько минут показывает результат в браузере.

После этого любой сотрудник открывает задачу и видит оба текста рядом — оригинал и перевод. Если нужно что-то поправить — правит прямо здесь. Если всё в порядке — нажимает «Опубликовать». Изменения уходят в английский репозиторий автоматически.

Руководитель в любой момент видит, что сейчас переводится, что уже опубликовано, кто и когда нажал кнопку.

### Из чего состоит сервис

Три части, которые работают вместе:

- **Бэкенд** — принимает уведомления от GitHub, ставит задачи в очередь, запускает пайплайн перевода
- **База данных** — хранит задачи, логи и правки редактора
- **Веб-интерфейс** — для просмотра статусов, редактирования и публикации

Один момент, который стоит отметить отдельно: пока идет перевод, логи появляются в браузере в реальном времени. Это важно, чтобы видеть прогресс и сразу замечать ошибки — не нужно ждать конца и разбираться в отчете постфактум.

### Как строить поэтапно

Каждый этап дает готовый рабочий результат:

1. Автоматический запуск перевода при изменениях в репозитории, логи в браузере
2. Просмотр оригинала и перевода рядом, правки в тексте
3. Кнопка «Опубликовать» — изменения уходят в английский репозиторий
4. Роли для разных участников команды, история публикаций, просмотр аналитики

## Улучшения пайплайна

Текущий пайплайн справляется с основной задачей. Есть три доработки, которые повысят качество и надежность перевода.

**StyleValidator.** Перед тем как отправить документ в перевод, модель проверяет его стиль и структуру. Это страховка: если оригинал написан с ошибками, переводить его сразу нет смысла.

**DictionaryFixer.** После перевода иногда остаются русские слова — обычно это специфические термины, которые модель не распознала. DictionaryFixer ищет такие слова по словарю и заменяет напрямую, без обращения к LLM.

**LLMFixer.** Если после DictionaryFixer в тексте все еще есть русские фрагменты — они отправляются в LLM отдельным запросом на доперевод. Это последний барьер перед тем, как файл попадет к редактору.

## Другие идеи

**Кэш переводов.** Если один и тот же абзац встречается в нескольких документах, переводить его каждый раз заново — лишние затраты. Кэш хранит уже переведенные фрагменты и возвращает их без обращения к API. Дополнительный эффект — единообразие терминологии по всей документации.

**Инкрементальный перевод.** Переводить не весь файл, а только те строки, которые изменились с прошлого раза. Особенно актуально при встроенном запуске через CI/CD — когда правок немного, перевод занимает секунды.
