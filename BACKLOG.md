# Backlog

## 26.03.2026

Начали разработку MVP инструмента для автоматического перевода `.md` файлов документации с русского на английский.

### Что сделали

**Настроили проект:**
Создали структуру папок, установили зависимости через `uv`, настроили виртуальное окружение.

**Реализовали пайплайн перевода:**
Написали пять модулей которые работают последовательно — читают файл, защищают структуру, переводят, проверяют результат и сохраняют.

- `parser.py` — читает `.md` файл, проверяет что файл существует и имеет правильное расширение
- `protector.py` — защищает элементы которые нельзя переводить: блоки кода, ссылки, битрикс-теги, названия полей API, якоря. Заменяет их на плейсхолдеры `[[PROTECTED_N]]` и восстанавливает обратно после перевода
- `translator.py` — отправляет текст в Bitrix GPT и возвращает перевод
- `validator.py` — сравнивает структуру оригинала и перевода: считает заголовки, таблицы, блоки кода, проверяет что все термины из словаря переведены
- `writer.py` — сохраняет результат в папку `output/` с суффиксом `.en.md`

**Настроили конфигурацию:**
Вынесли промпт в отдельный файл `prompt.txt` и словарь терминов в `dictionary.json` — оба редактируются без изменения кода. API ключи хранятся в `.env`.

**Сделали CLI:**
Скрипт запускается из командной строки. Поддерживает перевод одного файла и целой папки, а также режим `--dry-run` для проверки пайплайна без обращения к API.

**Добавили логирование:**
Каждый шаг пишется в консоль и в файл в папке `logs/` с временной меткой.

### Простыми словами

Этот скрипт делает: берёт файл, сам разбирается что переводить, а что не трогать, отправляет текст в ИИ, проверяет что ничего не сломалось и сохраняет готовый результат. Одна команда в терминале — и файл переведён.
