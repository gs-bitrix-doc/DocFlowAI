# Ideas & Roadmap

## Пайплайн следующего уровня

Полноценный production пайплайн после MVP:

```
Файл
  │
  ▼
StructureValidator          ← проверяем структуру до перевода
  │ ошибки? → стоп, логируем
  │
  ▼
StyleValidator (LLM)        ← соответствует ли документ стилю
  │ не соответствует? → стоп, логируем
  │
  ▼
Protector.protect()         ← защищаем структуру плейсхолдерами
  │
  ▼
Translator (LLM)            ← переводим
  │
  ▼
Protector.restore()         ← восстанавливаем структуру
  │
  ▼
DictionaryFixer             ← термин в словаре но не переведён → заменяем
  │
  ▼
LLMFixer (LLM)              ← остался русский текст не из словаря → LLM
  │
  ▼
StructureValidator          ← финальная проверка структуры
  │
  ▼
DockerValidator             ← сборка документации
  │ ошибки сборки? → помечаем NEEDS REVIEW
  │
  ▼
Writer                      ← сохраняем
```

### Новые модули

| Модуль | Что делает |
|---|---|
| `StyleValidator` | LLM проверяет стиль и качество документа до перевода |
| `DictionaryFixer` | Находит непереведённые термины из словаря и заменяет напрямую |
| `LLMFixer` | Отправляет оставшийся русский текст в LLM на доперевод |
| `DockerValidator` | Запускает YFM сборку через Docker, возвращает ошибки |

## Другие идеи

### Сегментация по секциям
Разбивать файл по заголовкам `##` и переводить каждую секцию отдельно.
Решает проблему с большими файлами которые не влезают в контекстное окно модели.

### Кэш переводов (Translation Memory)
Сохранять уже переведённые фразы/абзацы. Если тот же текст встречается
в другом файле — брать из кэша без запроса к API. Экономия токенов + единообразие.

### Инкрементальный перевод
Переводить только то что изменилось с прошлого раза через `git diff`.
Актуально для CI/CD — при коммите переводятся только изменённые строки.

### Параллельный перевод
Несколько файлов из батча переводятся одновременно через `asyncio`.
Ускорение при большом количестве файлов.

### Human-in-the-loop
После перевода сохранять в папку `review/`. Человек одобряет → файл
переносится в `output/`.

### Интеграция в CI/CD
Запуск скрипта при коммите через GitHub Actions или Bitrix CI.
Автоматический перевод при обновлении документации.

### Валидация качества документации
Проверка структуры, соответствия шаблону, полноты разделов —
до перевода и независимо от него.
